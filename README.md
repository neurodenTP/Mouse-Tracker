# Mouse Maze Tracker

Система для автоматического трекинга поведения мышей в лабиринте.



## Конфигурация

Все настройки задаются в config.yaml.


## Особенности сбоки

Сбока производится через консоль

pyinstaller build.spec

После сборки в папке dist/MouseTracker/ появится MouseTracker.exe. 

Необходимо вручную скопировать туда следующие файлы:

config.yaml — из корня проекта (обязательно!)

ImportData/ — папка с видео (если нужно)

ExportData/ — создайте пустую папку для результатов

Готовые exe релизы: https://github.com/neurodenTP/Mouse-Tracker/releases


## Выходные данные

ExportData/tracking_data.csv — метрики по кадрам

ExportData/tracking_result.mp4 — видео с визуализацией

ExportData/snapshots/ — PNG-снимки

ExportData/calibration.json — параметры калибровки

## Структура проекта

├── config.yaml             # Конфигурация эксперимента (параметры детектора, метрик, рендереров)

│   ├── setup.py            # Калибровка (ROI, образец мыши, контекст)

│   ├── metric.py           # Метрики (скорость, дистанция, паузы, тепловая карта)

## Основные классы

### Source (src/source.py)

Базовый класс для источников видео. Реализует открытие, чтение кадров и освобождение ресурсов.

- SourceCamera — захват с камеры устройства

- SourceFile — воспроизведение видеофайла с диска

Оба класса поддерживают обрезку ROI и получение FPS.

### Detector (src/detector.py)

Базовый класс для детекции объекта на кадре. Возвращает DetectionResult с центром, bbox, маской и уверенностью.

- DetectorStub — заглушка для тестов (возвращает фиксированную точку)

- DetectorColorThreshold — детекция по цветовому порогу в HSV с морфологической фильтрацией и слиянием контуров

- DetectorColorThresholdKalman — детекция DetectorColorThreshold с добавлением фильтра Калмана для сглаживания траектории.


### Visualizer (src/visualizer.py)

Базовый класс для отрисовки данных на кадре.

- VisualizerCenterPoint — рисует точку в центре объекта

- VisualizerContour — обводит контуры из маски детекции

- VisualizerHeatmap — накладывает тепловую карту посещений (получает данные из MetricHeatmap)

### Renderer (src/renderer.py)

Базовый класс для вывода результатов обработки.

- RendererDisplay — отображение в окне OpenCV

- RendererFile — запись видео в файл MP4

- RendererCSV — запись метрик в CSV (устойчиво к обрыву питания)

- RendererSnapshot — сохранение PNG-снимков каждые N кадров или в конце

### Metric (src/metric.py)

Базовый класс для расчёта метрик.

- SpeedMetric — мгновенная скорость (пиксели/секунду)

- DistanceMetric — накопленная дистанция

- PauseMetric — детекция пауз (время без движения)

- MetricHeatmap — накопление тепловой карты посещений

- MetricsTracker — контейнер для управления метриками и хранения истории кадров

### ConfigLoader (src/config_loader.py)

Загрузка конфигурации из YAML и фабрики для создания объектов. Позволяет создавать все компоненты на основе настроек из config.yaml без правки кода.

- create_source() — создаёт источник видео

- create_context() — проводит калибровку или загружает сохранённую

- create_detector() — создаёт детектор с применением калибровки

- create_metrics() — создаёт список метрик

- create_tracker() — создаёт трекер метрик

- create_visualizers() — создаёт список визуализаторов

- create_renderers() — создаёт список рендереров

### CalibrationContext (src/setup.py)

Контейнер для данных калибровки. Поддерживает сохранение и загрузку в JSON, автоматическое восстановление типов.

- save() — атомарное сохранение в JSON

- load() — загрузка с восстановлением типов

### Runner (src/runner.py)

Оркестратор цикла обработки. Открывает источник, читает кадры, вызывает детектор, метрики и рендереры.
